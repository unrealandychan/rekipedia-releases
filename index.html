Features

Everything your team needs
to understand any codebase

From a single binary to a full agentic wiki pipeline — rekipedia works with any LLM, any repo, any team.

🧠
Agentic wiki orchestration
PlannerAgent designs the wiki structure dynamically based on your repo — 3–15 pages, adapted to what's actually there.
🔍
Hybrid RAG Q&A
FAISS-indexed code chunks + wiki pages give the LLM full codebase context. Answers cite real file paths and line numbers.
⚡
Incremental updates
After the first scan, only changed files are re-processed via SHA-256 diffing. Wiki refreshes in seconds, not minutes.
🔌
Any LLM, any provider
Uses litellm under the hood — Ollama, OpenAI, Anthropic, Gemini, or any OpenAI-compatible endpoint. Fully local works too.
📦
Portable SQLite store
Everything lives in a single .db file. No server, no cloud dependency. Commit it, share it, or keep it gitignored.
📤
Wiki export
Bundle to a single Markdown file, ZIP archive, or structured JSON for downstream tooling and documentation pipelines.
🌐
Local web UI
Browse generated wiki pages in a dark-themed web interface. Ask questions via streaming Q&A — no API calls leave your machine.
🤖
AI agent skill
Ships a Hermes agent skill for Copilot, Claude Code, and Codex — dramatically reduces context window usage for large codebases.
📉
40–60% token reduction
Context slicing ensures each wiki page only receives the data it needs — far more efficient than fixed-layout approaches.
CLI Reference

Everything you need,
one command at a time

A focused set of commands that cover the entire lifecycle from init to interactive Q&A.

rekipedia init [repo]
Scaffold .rekipedia/ with config.yml and update .gitignore. Always run first.
reki scan [repo]
Full analysis — extracts symbols, synthesises 3–15 wiki pages via PlannerAgent, exports JSON manifest.
reki ask [question]
Interactive Q&A REPL with streaming answers. Add -q for single-shot mode. Ctrl+C to quit.
reki update [repo]
Incremental refresh — re-processes only changed files. Falls back to full scan if no previous run exists.
reki serve [repo]
Start a local web UI at http://127.0.0.1:7070 to browse wiki pages and ask questions via browser.
reki embed [repo]
Build or rebuild the FAISS semantic search index for hybrid RAG Q&A. Supports any litellm-compatible embed provider.
rekipedia export [repo]
Bundle the wiki to a single file. Supports --format md|zip|json for downstream pipelines.
Installation

Get started in seconds

Multiple install methods — pick what fits your workflow. No cluster access required.

# macOS & Linux — no Python or Node required
brew tap unrealandychan/tap
brew install rekipedia

# Get started
rekipedia init .
reki scan .
# Core (scan + serve + ask)
pip install rekipedia
# or with uv (recommended)
uv tool install rekipedia

# With RAG support (semantic search)
pip install "rekipedia[rag]"

# No install — run directly
uvx reki scan .
# No install required
npx rekipedia init .
npx reki scan .
npx reki ask "How does the auth flow work?"
# macOS Apple Silicon
curl -L https://github.com/unrealandychan/rekipedia-releases/releases/download/v0.9.3/reki_darwin_arm64.tar.gz | tar xz
sudo mv rekipedia /usr/local/bin/

# macOS Intel
curl -L https://github.com/unrealandychan/rekipedia-releases/releases/download/v0.9.3/reki_darwin_amd64.tar.gz | tar xz
sudo mv rekipedia /usr/local/bin/

# Linux x86_64
curl -L https://github.com/unrealandychan/rekipedia-releases/releases/download/v0.9.3/reki_linux_amd64.tar.gz | tar xz
sudo mv rekipedia /usr/local/bin/
After init, edit .rekipedia/config.yml
version: 1
llm:
  model: ollama/llama4        # any litellm model string
  api_key: ""                # or set CLOSE_WIKI_API_KEY
  base_url: ""               # for local / self-hosted
  temperature: 0.2
